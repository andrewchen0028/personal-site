Skydio - Autonomy & Control Internship

 Summer 2021

Skydio is the premier U.S. drone manufacturer and creates the most advanced autonomous drones in the world. I added two modules to Skydio's internal simulation stack, which is used for testing software before hardware integration. Both modules were written in Python and C++.

  • The first module adds collision physics for simulated objects. We wanted this functionality to be able to test scenarios with the vehicle taking off from and landing on raised platforms. This module involed algebraic operations on signed distance functions, as described on Inigo Quilez's site. The collision force vector itself was modeled using simple mass-spring damper dynamics.

  • The second module enables simulation of sensor drift in flight. Skydio drones use a suite of six cameras for visual-inertial odometry, which becomes prone to sensor drift when factors such as high altitude or rapid motion degrade the accuracy of the vehicle pose estimate. My module allows for granular spoofing of sensor drift in the simulator: the user can choose any combination of pseudorandom or user-supplied drift vectors, and set drift sensitivity to each of the factors which influence drift severity in real life.

Propulsive Landing Guidance Project

 Spring-Fall 2020

During quarantine, I worked with two classmates to implement a variant of the G-FOLD propulsive landing guidance algorithm in Python, based on "Lossless Convexification of Nonconvex Control Bound and Pointing Constraints of the Soft Landing Optimal Control Problem" (Açıkmeşe et. al.).

  • Heavily-modified build of Kerbal Space Program flight simulator, running kRPC mod to control a 1:1 model of a Spacex Grasshopper using external code.
  • Vehicle is manually piloted to 300m AGL with slight pitch offset from vertical.
  • Python script takes full control at ~40sec, uses ECOS embedded SOCP solver to generate thrust vectors for fuel-optimal landing.

Draper Laboratory: Flight Software Validation Internship

 Summer 2020

I created parallelized anomaly detection tools for Sierra Nevada's Dream Chaser spacecraft. These tools were written in MATLAB to scan over 50GB of data from Monte Carlo simulations and hardware-in-loop telemetry, with user-configurable thresholds on error magnitude and duration to identify anomalous data.
